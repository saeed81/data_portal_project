AeroBulk is a package/library that gathers state-of-the-art aerodynamic bulk formulae algorithms used to estimate turbulent air-sea fluxes in an efficient and unified way. These turbulent fluxes are wind stress, evaporation (latent heat flux) and sensible heat flux, they are needed as part of the surface boundary conditions of OGCMs, AGCMs and in the coupling interface of Earth Systems.

Based on the so-called bulk model (see Eq.1) and the constant flux layer approximation, AeroBulk can compute these turbulent fluxes based on the following iterative algorithms, each relying on the Monin-Obukhov similarity theory:
  • COARE algorithm version 3 (Fairall et al. 2003)
  • NCAR algorithm (Large & Yeager 2004, 2008)
  • ECMWF algorithm (IFS documentation)
Beside bulk algorithms AeroBulk also provides a set of functions to accurately estimate bulk transfer coefficients, relevant atmospheric variable such as density of air, different expressions of the humidity of air, viscosity of air, specific humidity at saturation, Monin-Obukhov length, wind gustiness, etc...

The focus in AeroBulk is readability, efficiency and portability towards either modern GCMs (Fortran 90, set of modules and a library) or diagnostic tools (C++ library and Python modules).

> Theory behind Aerobulk
    The Bulk Model
Based on the constant flux layer approximation, the turbulent fluxes of interest at the air-sea interface are estimated using the traditional bulk aerodynamic formulae. Each of these fluxes is expressed as proportional to a bulk transfer coefficient, the difference of the relevant quantity between a given height z above the surface and the sea surface, the density of air, and to what we refer to as the bulk wind speed.

> Computing transfer coefficients with AeroBulk
In AeroBulk, 3 different routines are available to compute the transfer coefficients CD, CH and CE:
  • turb_coare_2z() of module mod_blk_coare (mod_blk_coare.f90)
  • turb_ncar_2z() of module mod_blk_ncar (mod_blk_ncar.f90)
  • turb_ecmwf_2z() of module mod_blk_ecmf (mod_blk_ecmwf.f90)
> Computing turbulent fluxes with AeroBulk
Alternatively, AeroBulk can also directly compute the 3 turbulent fluxes with the routine aerobulk_model() of module aerobulk (aerobulk.f90): 

PROGRAM TEST_FLUX
  USE mod_const
  USE aerobulk
    ...
 CALL AEROBULK_MODEL(calgo, zt, zu, sst, t_zt, q_zt, U_zu, V_zu, SLP, &
      &              Qe, Qh, Tau_x, Tau_y,                            &
      &              Nitter=20)
    ...
END PROGRAM TEST_FLUX
INPUT ARGUMENTS:
  • calgo: name of the algorithm to use (coare/ncar/ecmwf)
  • zt : height for temperature and spec. hum. of air [m]
  • zu : height for wind speed (generally 10m) [m]
  • sst : SST [K]
  • t_zt : potential air temperature at zt [K]
  • q_zt : specific humidity of air at zt [kg/kg]
  • U_zu : zonal scalar wind speed at 10m [m/s]
  • V_zu : meridional scalar wind speed at 10m [m/s]
  • SLP : sea-level pressure [Pa]
OUTPUT ARGUMENTS:
  • Qe : latent heat flux [W/m^2]
  • Qh : sensible heat flux [W/m^2]
  • Tau_x : zonal wind stress [N/m^2]
  • Tau_y : meridional wind stress [N/m^2]
OPTIONAL ARGUMENT:
  • Nitter : number of itteration in the bulk algorithm
> Computing atmospheric state variables with AeroBulk
A selection of useful functions to estimate some atmospheric state variables of the marine boundary layer are available in the module mod_thermo (mod_thermo.f90).
Example for computing SSQ of Eq.(1) out of the SST and the SLP: 

PROGRAM TEST_THERMO
  USE mod_const
  USE mod_thermo
  ...

  SSQ(:,:) = 0.98*q_sat(SST, SLP)
  ...
END PROGRAM TEST_THERMO
